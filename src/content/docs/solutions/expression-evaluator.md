</style><svg style="display: none;"><defs id="MathJax_SVG_glyphs"></defs></svg><p>A Researcher has introduced a special type of mathematical expression as a part of one of his research. He has several numbers of, randomly generated expressions that he wanted to evaluate. But because of his buggy generation algorithm, those expressions can have invalid syntax. As a programmer, he has been asked you to help to test those algorithms.</p>

<p>You have to write a correct algorithm to evaluate the given list of expressions. If the expression is invalid you have to show the message "invalid" otherwise the value of the result.</p>

<p>The grammar of the expressions is explained below.</p>

<p>An expression can contain several numbers of binary operations, integers, parenthesis. The operators are given below, </p>

<ul>
<li>Addition <code>+</code>, Difference <code>-</code></li>
<li>Multiplication <code>*</code>, </li>
<li>Minimum <code>%</code>, Maximum <code>@</code></li>
</ul>

<p>Operators are ordered according to their precedence. +,- have same precedence. %,@ have the hightest precedence among all operators. Other than this expression inside the paranthesis should be evaluated first.</p>

<p>To avoid conflicts If there are multiple operation with same precedence following decisions should be taken,</p>

<ul>
<li>if there are multiple aditions, differences; then among them left most operator should be evaluated first.</li>
<li>if there are mutiple minimum, maximum operators; then among them the right most operator should be evaluated first.</li>
</ul>

<p>Also there are some guidelines to follow,</p>

<ul>
<li>All the operation are binary operations. There cannot be unary operation such as negative or positive.</li>
<li>It is guarenteed that the input expression does not contains any character other than these <code>digit</code>, <code>+</code>, <code>-</code>, <code>*</code>,<code>%</code>, <code>@</code>, <code>(</code>, <code>)</code>.</li>
<li>The <code>-</code> operator is special because it always results a positive value. As an example imagine that <code>a-b</code> is there and a &lt; b then answer must be <code>b-a</code>.</li>
<li>Integers cannot be start with zero if there is integers start with zero then the expression is invalid.</li>
<li>Expression cannot be empty.</li>
<li><code>%</code>, <code>@</code> operators results minimum and maximum respectively. As an example <code>45%32</code> will give <code>32</code>; <code>43%43</code> will give <code>43</code>.</li>
</ul>
